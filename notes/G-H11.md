# **在Windows环境下安装部署Hermes Agent**

在 Windows 上搭建 Linux 开发环境，Windows Subsystem for Linux (WSL) 无疑是最佳选择。本文将详细介绍如何安装 WSL 2、配置 Ubuntu 24.04 LTS，并将其迁移到非系统盘以节省 C 盘空间。

### **一、WSL2环境搭建**

### **启用核心功能**

通过Windows PowerShell（管理员模式）执行以下命令启用虚拟化支持：

1.  `# 启用WSL功能（需重启生效）`
    
2.  `dism.exe /online /enablefeature /featurename:MicrosoftWindowsSubsystemLinux /all /norestart`
    
3.  `# 启用虚拟机平台（WSL2必需）`
    
4.  `dism.exe /online /enablefeature /featurename:VirtualMachinePlatform /all /norestart`
    

**操作提示**：执行后需重启系统使配置生效，可通过`systeminfo | find "Virtualization"`验证虚拟化支持状态。

快速安装：使用 wsl --install 命令 对于 Windows 10 版本 2004 及更高版本或 Windows 11 用户，这是最简单的安装方法。

一键安装步骤 1、**以管理员身份打开 PowerShell**

2、**执行快速安装命令（wsl --install）**

3、**重启计算机**

## 遇到的问题：

1、WslRegisterDistribution failed with error: 0x800701bc Error: 0x800701bc WSL 2 ?????????????????? [https://aka.ms/wsl2kernel](https://aka.ms/wsl2kernel)

原因：**WSL 2 缺少了运行所需的 Linux 内核**

解决：运行以下代码更新安装（管理员身份运行PowerShell） `wsl --update`

2、PS C:\\WINDOWS\\system32> wsl --install -d Ubuntu 正在下载: Ubuntu 服务器返回无效或不可识别的响应 % \] 错误代码: Wsl/InstallDistro/0x80072f78

原因：这个 `0x80072f78` 错误通常表示 WSL 在下载所需的内核或系统文件时，网络连接出现了问题

解决：检查并重新配置 Windows 功能没问题后，再次执行该指令
