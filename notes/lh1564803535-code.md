# 2026-05-20

x402 规定了 Agent 怎么付款，但没规定：

-   人给 Agent 授权多少额度？在哪里设定？
    
-   Agent 付了什么人能看到？多清楚？
    
-   Agent 要超限了，怎么冒泡问人？
    
-   出错了，"收据"（Signed Receipt）怎么变成人能读懂的纠纷证据？
    

这四个问题是纯 UX 问题，不是协议问题。协议开发者不擅长，评委最关注。

## **最关键的一个扩展：**`upto` **scheme**

这是你要重点看的。不是 `exact`（固定价格），而是：

> **买方预授权一个上限，卖方按实际用量收费。**

像打表计费的出租车——你上车前说"最多给 $10"，下车按实际里程付。

这意味着 Agent 在执行任务过程中，可以动态消耗服务，不用每次问人。但"$10 上限"是谁设的？在哪设的？能不能针对不同类型的服务设不同上限？能不能临时提额？**这是你作为 UX 设计师要设计的核心界面。**

## **还有一个宝：Bazaar（发现层）**

x402 文档里有个 Bazaar 扩展——Agent 怎么"找到"可用的付费服务。机器可读的服务目录，像 App Store 但给 Agent 看的。

**产品想象**：一个 Agent，预算 $5，目标是"帮我研究某个链上项目"。它自主去 Bazaar 找合适的数据服务，比价，付最便宜的，完成任务，给你一张人话收据。人只做一件事：设初始预算。

这就是 Agentic Commerce 的完整闭环，4 周能做出来的最小版本。

![Snipaste_2026-05-20_17-31-15.png](https://raw.githubusercontent.com/IntensiveCoLearning/AI-Web3-School/main/assets/lh1564803535-code/images/2026-05-20-1779282340893-Snipaste_2026-05-20_17-31-15.png)

ERC-8004 讲的是 Agent 身份和信誉。你带一个具体问题去：

> **"Agent 历史上在 x402 上付过多少钱、付给谁，这个记录能不能作为信誉依据？一个'花钱靠谱'的 Agent 和一个新 Agent，人给他们的预授权上限应该不同吗？"**

这个问题把 x402（支付）和 ERC-8004（信誉）连起来，是 Agentic Commerce 里最有意思的交叉点。课上能问出这个，你在组队时就有了清晰的产品角度。

这几天使用下来AI的感受就是没有用AI来提效是为了消耗token而去消耗token，一个好的想法比啥都重要。**交易所钱包** ：用户充值时交易所会提供专属充值地址，资金会归集到热钱包，提现由热钱包转出，地址创建基于随机数，由 SDK 实现。

-   **AI 在安全方面的应用** ：可用于分析未验证的合约代码，如去年很多未验证合约通过 AI 分析；Cloud Code 的 Misos 是 AI 帮忙做安全的工具。
    
-   **交易所订单簿数据** ：大概率可通过 API 获得，流动性提供商通过 API 监控价格和套利，但无法解决插针问题。
    
-   **量化数据处理** ：量化策略多为内部知识，大数据在 Web3 的应用需求有但不多，Web3 与 Web2 有共性，资金体量相对较小。
    
-   **链上小额免密支付** ：存在信任问题，用户可能不愿授权，可通过账号权限拆分解决，如 MPC 方案。
    
-   **钱包监听交易** ：可通过轮询或 Websocket 实现，本质上还是 HTTP 请求；可参考开源项目 epust 做多链监听索引。
    
-   **AA 钱包** ：会越来越成熟，以太坊有融合计划，但不同链的实现细节不同，多链生态中 NPC 可能更好。
    
-   **密码学学习程度** ：以 MPC 签名为例，要搞明白私钥切割计算方式、每轮交互内容，进一步考虑 ZK 证明防止作恶。
    
-   **跨链方向** ：需求会增多，但目前不算大，熊市业务受影响，牛市时链间活跃度可能提升；目前没有完全纯去中心化的跨链方案。
    
-   **面试基础技术岗** ：会问基础知识和 AI agent 使用情况，一面多问基础，二面、三面可能给设计稿和场景题，基础好易被识别，态度要诚恳。
    
-   **区块链监听** ：监听链上成本低，可付费订阅并进行双重校验，自行维护成本高。
    
-   **智能合约审计找工作** ：建议参考接力，要会相关知识并能证明，可学习密码学大神 lindell 的课程。
    
-   **AI coding 交易辅助工具放权边界** ：本质是安全边界，可参考风控规则，如短时高频、偶发大额深夜交易等，风控通过自动处理，不通过则人工辅助签名。
    
-   **WEB3 招聘平台** ：可关注项目方，直接向项目方投递简历，部分老外团队要求不高。
    
-   **Uniswap V4 hook 发展前景** ：分享者未了解，无法给出建议。
    
-   **Agentic payment X402 产业** ：Agentic payment 是信任问题，不算新技术；X402 前段时间火，但实际应用少。然后就是听老师的学习到的新东西。
