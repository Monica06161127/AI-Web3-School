1\. AI Agent 与普通大模型对话的根本区别在于对工具的调度能力。Hermes能够极其精准地理解系统指令，输出格式化的 JSON 数据来调用外部 API。它就像是一个“大脑”，通过连接外部的计算器、搜索引擎或区块链数据接口，真正实现了从单纯的聊天到解决实际问题

2\. 准在云端环境跑通第一个 Agent 闭环 结合现有的 Linux 云服务器和 Python/Streamlit 部署经验，我准备通过 API 接入或本地拉取一个小参数的 Hermes 模型，写一个基础的 Python 脚本来测试它的 Function Calling 能力。初步设想是让它自动调用接口，抓取并结构化处理一段金融市场的特定行情数据，真正在自己的服务器上把 Agent 从 0 到 1 跑通一次。

# 2026-05-18

-   **AI 在 Web 开发中的作用**：
    
    -   **设计协助**：AI 可协助人进行设计细化和分析，作为独立专家梳理未想到的数据流。
        
    -   **编码验证**：AI 自我验证在 AI 编码中非常重要，有助于实现高质量编码。
        
-   **Web2 与 Web3 项目能力及安全特性**：
    
    -   **项目能力占比**：单项目里 Web2 能力占比较高，常见应用如币安、policy market 等，Defi 项目如 uniswap、dydx 等离不开 Web2 前端页面。
        
    -   **安全特性差异**：Web2 支付体系需信任权威，资金通过银行卡账户和支付密码控制；Web3 资金交互有其特性，安全源于扎实基础和架构设计。
        
-   **Web3 支付流程及安全考量**：
    
    -   **支付流程**：支付服务先获取支付单信息展示给用户，法币支付与 cript 有所不同，cript 因使用与 Blockchain 相同算法无需额外请求。
        
    -   **安全考量**：需考量 API 安全，用户与电商平台、支付服务间的信任问题，可通过私钥签名校验保证数据不可篡改，但私钥泄露会降低信任。
        
-   **钱包相关概念及分类**：
    
    -   **钱包角色**：钱包在 Web3 中扮演交易确权角色，签名分交易签名和消息签名，可证明意图和身份。
        
    -   **钱包分类**：包括 EOA 钱包、合约钱包；按托管方式分全托管钱包、混合托管钱包；还有硬件钱包、隐私钱包等。
        
    -   **私钥问题**：私钥泄露或丢失后果严重，多签可将私钥切片，MPC 和合约钱包可实现多签。
        
-   **钱包使用的风险与场景**：
    
    -   **风险问题**：存在私钥泄露、误签交易和消息、权限滥用等风险，EIP712 签名可视化仍不够完善。
        
    -   **使用场景**：钱包在交易生命周期中用于构造交易、签名交易并广播，关键参数包括 gas fee、nonce、instruction 和 data 等。
        
-   **交易监听与确认**：
    
    -   **监听逻辑**：支付服务确认交易是否收到，需关注链上地址，有些交易需等待 finalize 时间。
        
    -   **风险防控**：可通过交易可视化、模拟等方式降低风险，还可通过 EUA、MPC 加密私钥完成签名。
        
-   **学习路径与相关问题**：
    
    -   **学习建议**：安全学习要自己能写作业、判断链的基本东西，再有的放矢学习。
        
    -   **钱包管理问题**：浏览器插件钱包交易管理、交易所充换钱包地址及交易归集逻辑等问题被提及。
        
    -   **AI 应用探讨**：探讨了 AI 在 Web3 安全方面的应用，提醒不要焦虑 AI，要扎实基础并利用好 AI。
