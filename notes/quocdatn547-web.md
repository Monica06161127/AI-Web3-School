# 2026-05-21
<!-- DAILY_CHECKIN_2026-05-21_START -->
一，AI和Web3结合的原因：web3是去中心化的，而AI是中心化的。算力资源掌握在少数公司手里.

**两者结合的愿景：** 如果能用 Web3 的去中心化方式来承载 AI 的算力、数据和服务，那 AI 就不再属于某一家公司，而是属于全世界。

二，web3给AI 赋能的五个方向：

1.去中心化 AI 基础设施

2.AI Agent + 钱包

3.AI 驱动的链上分析工具

4.智能钱包和安全助手

5.交易所和 DeFi 智能风控系统

我个人感觉，第 3 到第 5 点确实现在就可以用上，而1，2 点距离实际应用还是有点遥远。

# 2026-05-20
<!-- DAILY_CHECKIN_2026-05-20_START -->

——第一感觉就是web3的体系很庞大，很复杂。

回到正题：

1.我学到了一笔交易从发出到最后确认，需要经历：Wallet（签名）→ RPC/Node（传播）→ Mempool（排队）→ Builder（排序打包）→ Validator（出块/证明）→ Block（可查询）。

2.私钥相当重要，泄露直接完蛋。（联想到做网站时，自己的api key也不能泄露）

3.真正的账本和数据都在链上，所以可以用不同的钱包软件管理同一个账户。（还挺方便的哈）

4.比较有压力的是，里面的概念非常多，比如 Gas、Nonce、Call Data、RPC、Mempool、Validator，有的学了啊。今天先接受自己是小白，慢慢学吧
